MAKI POS — Current UI · Modals & Bottom Sheets (cross-cutting capture)

This is not one screen — it is every overlay surface in the app. The goal of this hand-off is to design ONE synchronized modal + bottom-sheet language that all showDialog/AlertDialog and all showModalBottomSheet/DraggableScrollableSheet surfaces share. Today these are a patchwork: raw AlertDialogs that inherit the theme's 24px radius sit next to custom Dialogs that pin their own radii and colors; bottom sheets each hand-roll their own grab handle, padding and footer; icons are mostly Cupertino (one dialog already migrated to Lucide); there is no shared confirm / destructive / input component; and the password dialog exists twice. The frames below group the live UI by archetype (not by screen) so the designer can define a single shell + a small set of variants. Light theme is shown — dark parity is required (both themes are defined in app_theme.dart). Mark up the redesign and hand back. (Icons here render in Lucide — the migration target — even though most live surfaces still ship Cupertino glyphs.)

Bottom sheets: mobile/widgets/drafts/draft_detail_sheet.dart (DraggableScrollableSheet 0.7/0.5/0.95), mobile/screens/sales/void_requests_screen.dart resolve sheet (DraggableScrollableSheet 0.85), mobile/widgets/inventory/stock_adjustment_dialog.dart (named "dialog" but is a showModalBottomSheet form), mobile/widgets/inventory/product_image_uploader.dart (image-source picker), mobile/screens/settings/settings_screen.dart (theme radio picker).

Theme defaults (lib/core/theme/app_theme.dart · light ~L259 / dark ~L597): dialogTheme = bg lightBackground #FFFFFF / darkCard, elevation 1, shadow black12/black54, surfaceTint transparent, shape radius AppRadius.xl = 24, title headingSmall, content bodyMedium. bottomSheetTheme = same bg, elevation 1, surfaceTint transparent, top radius AppRadius.xl = 24. AppRadius tokens: sm 10 · md 14 · field 16 · lg 18 · hero 22 · xl 24 · pill 999. Open question for the designer: dialogs & sheets currently use xl = 24; decide whether to keep 24 or move to hero = 22 / lg = 18 for parity with the redesigned AppCard surfaces. Scrim ≈ rgba(17,28,29,.32). Migration target = the global theme in design/handoff/maki-theme/ (Lucide, soft-shadow surfaces, AppColors status semantics with dark parity). Snackbars (showSuccess/Error/WarningSnackBar) are adjacent feedback, not modals — keep that pattern; do not fold it into the dialog shell.
