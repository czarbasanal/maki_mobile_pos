MAKI POS — Current UI · 09 · Drafts

Pre-redesign (mixed Material). The mobile Drafts flow lets a cashier save a cart-in-progress and resume it later. It has two routed screens — the Saved Drafts list (/drafts) and the Draft editor (/drafts/:id) — plus a tap-through detail bottom sheet. The list rows and sheet are already on soft-shadow Cards with theme-aware hairlines, but every icon is still Cupertino (doc, cart-badge-plus, trash, wrench, clock, pencil…) and the editor's item rows are raw Material Cards; nothing uses AppCard yet. There are no status color semantics — drafts have no state beyond "active". The one already-Lucide piece is the shared MechanicPicker dropdown. This bundle migrates the surface onto the elevated theme — Lucide + soft-shadow AppCard + neutral discipline with dark parity. Currency renders grouped (₱1,234.00); SKUs in Roboto Mono. Mark up the redesign you want and hand back. (Icons here render in Lucide — the migration target — even though the live screens still ship Cupertino.)

Token-accurate reconstruction of the current Drafts surface. Sources: lib/presentation/mobile/screens/drafts/drafts_list_screen.dart, lib/presentation/mobile/screens/drafts/draft_edit_screen.dart, lib/presentation/mobile/widgets/drafts/draft_list_tile.dart, lib/presentation/mobile/widgets/drafts/draft_detail_sheet.dart (+ shared MechanicPicker, SummaryRow, EmptyStateView/LoadingView). Entity: DraftEntity (name, items[] = SaleItemEntity, laborLines[], mechanicId/mechanicName, discountType, notes, createdBy/createdByName, createdAt/updatedAt; derived subtotal / totalDiscount / laborSubtotal / grandTotal / totalItemCount). Current state: list rows + detail sheet already on soft-shadow Cards with theme-aware hairlines & muted fills, but all icons are Cupertino (doc_text, cart_badge_plus, trash, wrench, clock, pencil, xmark, refresh) and the editor's item rows are raw Material Cards — no AppCard. Drafts have no status so there is no color semantics; the only accent is the slate primary (total, qty badge, service-job badge, Load button) and success-green for discount/labor affordances. Behavior to preserve: Load is destructive — loading a draft into the cart immediately deletes it from the collection (re-saving creates a new entry); if the cart is non-empty it prompts Replace Cart? first. Delete needs creator-or-admin permission (trash hidden otherwise) and confirms with an item/total preview + "cannot be undone". The editor persists labor/mechanic edits through the full updateDraft path (never updateDraftItems, which would drop labor); Checkout and Edit in POS both consume the draft then navigate. Totals math: grandTotal = subtotal − discount + laborSubtotal; labor lines require a fee > 0. Currency grouped ₱1,234.00; dates MMM d, h:mm a (list) / MMM d, y • h:mm a (editor). Migration target = the global theme in design/handoff/maki-theme/ (Lucide, soft-shadow AppCard, neutral-by-default discipline, dark parity).
