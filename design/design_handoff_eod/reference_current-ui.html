MAKI POS — Current UI · 06b · End-of-Day + Closing History

Pre-redesign (mostly Material). The cash-drawer closing flow — the End-of-Day review/close form, its closed read-only state, and the Closing History list — is largely raw Material: Card sections, TextFormField inputs with filled InputDecoration, an ExpansionTile history list, and a red destructive FilledButton. It is theme-aware for status colors (success-green "closed", warning-amber post-close / "over" variance, error-red "short" variance + Close button) but has no Lucide icons and no soft-shadow AppCard. Currency renders ungrouped-symbol (₱1,234.00) via toCurrencyWithoutSymbol(). This bundle migrates all three states to Lucide + AppCard while keeping the variance/closed/post-close color semantics + dark parity. Mark up the redesign you want and hand back. (Icons here render in Lucide — the migration target — even though the live screens still ship Cupertino.)
