Pre-redesign (raw Material). The admin User Management surface is plain Material: a summary row of tinted boxes, an app-bar role filter (PopupMenuButton) + show/hide-inactive toggle, a ListView of Material Cards (one per user) with a tinted CircleAvatar, a pill role badge, a You tag for the current user, an Inactive tag + strikethrough + 0.6 opacity for deactivated users, and a 3-dot overflow menu (Deactivate / Reactivate). The create/edit form is a TextFormField stack with a big role-tinted avatar, a custom radio-style role picker (3 cards), and (create-only) two obscured password fields. Cupertino icons throughout, no AppCard, no Lucide. Role colors are hard-coded Material swatches: admin = purple, staff = green, cashier = orange (not theme tokens). This bundle migrates it onto the elevated theme — Lucide + soft-shadow AppCard + theme-aware role/status colors with dark parity. (Icons here render in Lucide — the migration target — even though the live screen still ships Cupertino.)

Business guards (last-admin, self-demote, self-deactivate) are enforced in the use-case layer and surfaced as the inline error box / snackbars.
