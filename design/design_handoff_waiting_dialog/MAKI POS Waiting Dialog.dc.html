Cross-cutting · Waiting overlay

MAKI POS — waiting dialog

The blocking busy overlay every screen shows while an async action is in flight — saving a job order, billing out, deleting, syncing, exporting a report. One centered card on the elevated theme (radius 24, the same scrim + shadow as every dialog), a primary progress ring, and a verb-first label. Neutral by default — the ring uses the slate / gold primary, never a status color. It blocks input and is not tap-to-dismiss; only the cancelable variant offers a way out. Maps to the app's LoadingOverlay / showDialog(barrierDismissible:false). Full dark parity (gold leads).
