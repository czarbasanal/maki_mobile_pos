MAKI POS — Current UI · 10 · Settings

Pre-redesign (raw Material). The Settings stack is the largest bundle: a hub of grouped Card + ListTile sections (My Profile / Administration / General) with a profile hero + role pill, plus four admin sub-screens — Manage Lists (one tile per category kind), the per-kind Category editor CRUD, Cost Code Settings, the Mechanics editor — and a standalone About screen. Everything is plain Material: Card/ListTile/Divider, Cupertino icons, native Switch, AlertDialog forms, a theme-picker showModalBottomSheet, and password-gated cost-code edits. No AppCard, no Lucide. The role pill colors today are admin = red, staff = blue, cashier = green (note: spec proposes admin = purple). This bundle migrates onto the elevated theme — Lucide + soft-shadow AppCard with dark parity, neutral-by-default. (Icons here render in Lucide — the migration target — even though the live screens still ship Cupertino.)

Role gating: the entire Administration section (User Management, Activity Logs, Cost Code Settings, Manage Lists, Mechanics) renders only when currentUser.role == admin; My Profile + General show for all roles. Role pill colors today = admin red / staff blue / cashier green (spec proposes admin purple). Destructive / gated actions to preserve: deactivate (archive) toggles for category/mechanic rows; Cost Code save & reset are password-gated (PasswordDialog) and log activity; Reset to Default shows a confirm AlertDialog first. The hub has a // TODO Store Information row (no-op today) and a native showAboutDialog launched from the About hub row — separate from the richer standalone AboutScreen shown in frame 9. Currency grouped ₱1,234. No sign-out lives in Settings today. Migration target = the global theme in design/handoff/maki-theme/ (Lucide, soft-shadow AppCard, theme-aware colors with dark parity, neutral discipline).
